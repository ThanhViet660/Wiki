🚀 Cài đặt Plugin

Hướng dẫn cài đặt VietCraftUpgrade lên server

Cài đặt các plugin yêu cầu

  • Tải và cài NightCore (framework cốt lõi)
  • Tải và cài MMOItems (hệ thống vật phẩm)
  • (Tùy chọn) Cài VietStorage nếu muốn tích hợp kho

Tải VietCraftUpgrade

Đặt file VietCraftUpgrade.jar vào thư mục plugins/ của server.

Khởi động server

Restart hoặc start server. Plugin sẽ tự tạo thư mục config và dữ liệu mẫu.

Kiểm tra log

Tìm dòng: [VietCraftUpgrade] VietCraftUpgrade enabled! Loaded X groups.

💡 Plugin tự tạo group mẫu "weapons" với category "fire_sword" để bạn tham khảo cấu trúc.
⚙️ Thiết lập ban đầu

Cấu hình tỉ lệ thành công

Mở config.yml → đặt Success_Rate (khuyến nghị 100 cho testing).

Cấp quyền cho người chơi

Dùng plugin Permission (LuckPerms,...) cấp quyền:

  • vietcraftupgrade.command.open — mở menu trao đổi
  • vietcraftupgrade.command.upgrade — nâng cấp
  • vietcraftupgrade.command.select — xem danh sách nâng cấp

Tạo vật phẩm trong MMOItems

Đảm bảo Type và ID trong recipe trùng với MMOItems đã tạo. Nên theo quy ước đặt tên: ITEMNAME_LEVEL

Test hệ thống

Chạy /vcu open để mở menu và kiểm tra recipe hoạt động đúng.

🛠️ Hướng dẫn Editor GUI

Tất cả thao tác quản lý hệ thống trao đổi qua GUI trong game

ℹ️ Mở Editor bằng lệnh /vcu editor. Cần quyền vietcraftupgrade.command.editor
📁 Tạo Group mới

Mở Editor → Group List

Chạy /vcu editor → chọn mục Danh Mục Tổng (Group List).

Tạo Group mới

Click nút Tạo Mới. Nhập ID cho group trong chat (ví dụ: weapons, armors).

Đổi tên Group

Click vào group đã tạo → chọn Đổi Tên. Nhập tên hiển thị (hỗ trợ MiniMessage).

⚠️ ID group phải là chữ thường, không dấu, không khoảng trắng. Đây là ID nội bộ, không phải tên hiển thị.
📋 Tạo Category mới

Chọn Group chứa Category

Trong Group List, click vào group muốn thêm category.

Tạo Category

Click Tạo Category Mới. Nhập ID (ví dụ: fire_sword).

Cấu hình Category

  • Đổi tên: tên hiển thị trong menu (MiniMessage)
  • Layout: chọn layout menu riêng (hoặc để default)
  • Di chuyển: chuyển category sang group khác hoặc thành standalone
💡 Category cũng có thể tạo dạng standalone (không thuộc group nào). Chọn "Standalone" khi được hỏi group.
🧪 Tạo Recipe mới

Mở Category trong Editor

Vào Editor → Group → Category → chọn Danh sách Recipe.

Tạo Recipe

Click Tạo Mới. Dialog sẽ hỏi:

  • Recipe ID — tên nội bộ (ví dụ: fire_sword_lv2)
  • Result Type — MMOItems Type (SWORD, BOW...)
  • Result ID — MMOItems ID (FIRESWORD_2)

Cấu hình Recipe

  • Đổi tên: tên hiển thị cho người chơi
  • Thay đổi Result: đổi Type/ID/Amount kết quả
  • Slot/Page: đặt vị trí cố định trên menu
🧱 Quản lý Ingredients

Mở danh sách Ingredient

Trong Recipe Editor → chọn Nguyên Liệu.

Thêm Ingredient

Click Thêm Mới. Nhập:

  • Type — SWORD, MATERIAL, VANILLA,...
  • ID — ID vật phẩm (MMOItem ID hoặc Material name)
  • Amount — Số lượng cần
  • VietStorage — Bật/tắt lấy từ kho

Sửa Ingredient

Click vào ingredient → chỉnh sửa Số lượng, Tên tùy chỉnh, VietStorage toggle.

Xóa Ingredient

Click vào ingredient → chọn Xóa.

⚡ Auto-Create Recipes

Tạo hàng loạt recipe nâng cấp tự động

Mở chức năng Auto-Create

Trong Category Editor → chọn Auto Create Recipes.

Nhập thông tin

  • Result Type: loại MMOItem output (ví dụ: SWORD)
  • Result Base ID: tên gốc (ví dụ: FIRESWORD)
  • Max Level: số level tối đa
  • Ingredient Type: loại nguyên liệu chung (ví dụ: MATERIAL)
  • Ingredient ID: ID nguyên liệu (ví dụ: UPGRADE_STONE)
  • Ingredient Increment: số lượng tăng mỗi level

Kết quả

Plugin tự tạo chuỗi recipe. Ví dụ với max=3, increment=5:

  • Level 1: cần 5 UPGRADE_STONE
  • Level 2: cần 10 UPGRADE_STONE + FIRESWORD_1
  • Level 3: cần 15 UPGRADE_STONE + FIRESWORD_2
ℹ️ Từ level 2 trở lên, plugin tự thêm ingredient là vật phẩm level trước đó.
🎨 Tùy chỉnh Layout Menu

Tạo file layout mới

Copy menu/category/default.yml → đổi tên (ví dụ: premium.yml).

Chỉnh sửa layout

Mở file layout và chỉnh cấu trúc menu: kích thước, icon decoration, slot positions...

Gán layout cho Category

Có 2 cách:

  • Editor: mở Category → đổi Layout → chọn layout mới
  • File YAML: set Layout: "premium" trong file category

Reload

Chạy /vcu reload để áp dụng layout mới.

📌 Slot & Page tùy chỉnh

Mỗi recipe có thể được đặt vào slot và page cố định thay vì tự động sắp xếp.

SettingGiá trịMô tả
Slot: -1AutoPlugin tự sắp xếp vào slot trống
Slot: 0-53FixedĐặt cố định vào slot chỉ định (0-indexed)

💡 Dùng Recipe Layout Editor trong Editor GUI để kéo thả recipe vào vị trí mong muốn thay vì chỉnh file.
🎮 Hướng dẫn Người chơi

Cách trao đổi vật phẩm

Mở menu trao đổi

Gõ /vcu open để mở menu chính. Chọn Group → Category.

Xem recipe

Mỗi recipe hiện: tên, nguyên liệu (đủ ✔ / thiếu ✘), tỉ lệ thành công.

Trao đổi

  • 🖱️ Chuột trái: trao đổi 1 lần
  • ⬆️ Shift + Trái: trao đổi nhiều lần (theo số lượng đã chọn)
  • 🖱️ Chuột phải: xem chi tiết nguyên liệu

Nhận kết quả

Thành công → nhận vật phẩm + message. Thất bại → nguyên liệu mất hoặc trả lại (tùy config).

⬆️ Nâng cấp nhanh

🗡️ /vcu upgrade

  1. Cầm vật phẩm MMOItem trên tay
  2. Gõ /vcu upgrade
  3. Plugin tự tìm recipe nâng cấp
  4. Mở menu chi tiết để trao đổi

📋 /vcu select

  1. Gõ /vcu select
  2. Plugin quét toàn bộ inventory
  3. Hiện danh sách item có thể nâng cấp
  4. Click để xem chi tiết recipe
🔢 Shift Exchange

Chọn số lượng

Trong menu category hoặc detail: Shift + Right Click để mở dialog chọn số lượng.

Nhập số lượng

Nhập số lượng muốn trao đổi (ví dụ: 5, 10, 20). Số này được lưu vĩnh viễn cho player.

Trao đổi nhiều

Shift + Left Click trên recipe để trao đổi theo số lượng đã chọn. Plugin tự dừng khi hết nguyên liệu hoặc đầy inventory.

🔧 Xử lý lỗi thường gặp

❌ "Vật phẩm này không phải MMOItem!"

Nguyên nhân: Vật phẩm trên tay không phải MMOItem hoặc MMOItems chưa nhận diện.
Giải pháp: Đảm bảo item được tạo qua MMOItems, kiểm tra Type và ID trong /mi browse.

❌ "Không tìm thấy công thức nâng cấp!"

Nguyên nhân: Không có recipe nào có Result Type/ID trùng với level tiếp theo.
Giải pháp: Kiểm tra ID format (ITEMNAME_LEVEL), tạo recipe có Result ID đúng.

⚠️ Menu hiển thị vật phẩm lỗi

Nguyên nhân: MMOItems Type/ID trong recipe không tồn tại.
Giải pháp: Kiểm tra Type và ID bằng /mi browse <type>. Đảm bảo mọi recipe trỏ đến item có thật.

⚠️ Nguyên liệu VietStorage không được tính

Nguyên nhân: AllowStorage chưa bật, VietStorage chưa cài, hoặc type không phải VANILLA.
Giải pháp: Đặt AllowStorage: true, cài VietStorage, chỉ dùng cho type VANILLA.
