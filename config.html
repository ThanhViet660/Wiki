⚙️ config.yml

Cấu hình chính của plugin

📄 config.yml
# ═══════════════════════════════════ # Cài đặt hệ thống trao đổi # ═══════════════════════════════════ Exchange: # Tỉ lệ thành công cho mọi trao đổi (0-100) # Đặt 100 để luôn thành công Success_Rate: 100.0 # Có mất nguyên liệu khi trao đổi thất bại? # true = mất nguyên liệu # false = trả lại nguyên liệu Lose_Ingredients_On_Fail: true # Hiện dialog xác nhận trước khi trao đổi Confirm_Before_Exchange: false

🎲 Success_Rate

Giá trị: 0.0 đến 100.0
Mặc định: 100.0 (luôn thành công)

Công thức: Math.random() * 100 < successRate
Người có quyền bypass.successrate luôn = 100%

📦 Lose_Ingredients_On_Fail

true: Nguyên liệu bị mất khi thất bại → tăng độ khó
false: Nguyên liệu trả lại → gameplay friendly

Khi false: mỗi ingredient được tạo lại ItemStack và add vào inventory

✅ Confirm_Before_Exchange

true: hiện confirmation dialog trước khi trao đổi
false: trao đổi ngay khi click

Nên bật khi Success_Rate < 100 để người chơi xác nhận

🎨 Layout Menu

Hệ thống layout cho category menu

Cách hoạt động

  • File menu/category/default.yml là layout mặc định cho tất cả
  • Tạo thêm file .yml trong menu/category/ → thêm layout mới
  • Trong category YAML, đặt Layout: "tên_file" (không có .yml)
  • Nếu category ID trùng với tên file layout, tự động dùng layout đó
  • Nếu Layout trống, tìm layout theo category ID → nếu không có dùng default
📋 Ưu tiên chọn layout
# Thứ tự ưu tiên khi chọn layout cho category: 1. Layout field trong YAML # Layout: "custom_layout" 2. Category ID match # category ID = file layout 3. default.yml # Fallback mặc định
